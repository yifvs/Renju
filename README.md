# ⚫ 五指棋

一个基于 **Alpha-Beta 剪枝 + 杀棋模块** 的五子棋人机对战游戏，自带 **禁手裁判系统** 和 **棋手评级系统**，所有计算在浏览器本地完成，无需联网。

## AI 引擎详解

### 决策流程

```
+----------------------------------------------------------+
|                    AI 决策流程                            |
+----------------------------------------------------------+
|                                                          |
|  Step 1: 查开局定式库(含禁手过滤)                          |
|  +-- 当前局面精确匹配棋谱 --> 秒出应手                     |
|  +-- 不匹配 --> 继续                                      |
|                                                          |
|  Step 1B: AI执白专用策略池(加权随机)                       |
|  +-- 若AI=WHITE 且 手数<=12                               |
|  |   +-- 命中策略池 --> 加权随机选走法,每局不同             |
|  |   +-- 未命中 --> 继续                                   |
|  +-- AI=BLACK或手数>12 --> 跳过                            |
|                                                          |
|  Step 2: 检查必胜/必防(含禁手过滤+多威胁检测)               |
|  +-- 有连五/活四 --> 立即走棋                              |
|  +-- 对手有活四/冲四 --> 必须防守                          |
|  +-- 对手有多重冲四(>=2独立威胁) --> 标记多威胁            |
|  +-- 对手有活三且自己没活四 --> 建议防守                   |
|                                                         |
|  Step 3: 杀棋模块检测(含禁手过滤)                         |
|  +-- 有活三以上的进攻机会 --> 强制搜索16层确认杀棋         |
|  |   +-- 杀棋成立 --> 返回必胜走法                        |
|  |   +-- 杀棋不成立 --> 继续搜索                          |
|  +-- 无显著威胁 --> 继续                                  |
|                                                          |
|  Step 4: 迭代加深 Alpha-Beta 搜索(含禁手过滤+随机化)       |
|  +-- 2层 -> 4层 -> 6层 -> 8层 -> 10层 -> 12层(地狱模式)   |
|  +-- 每层: 走法排序 -> Alpha-Beta剪枝 -> 置换表缓存        |
|  +-- 近优走法随机选取(非必胜/必败局面)                     |
|  +-- 达到时间限制 --> 返回最佳走法                         |
|                                                          |
+----------------------------------------------------------+
```

### 搜索优化技术

| 技术 | 文件 | 效果 |
|------|------|------|
| **Alpha-Beta 剪枝** | `search.js` | 跳过无用分支，搜索效率提升数倍 |
| **走法排序** | `search.js` + `evaluate.js` | 进攻分×1.0 + 防守分×1.1 + 位置分，好的排序让剪枝更高效 |
| **杀手走法** | `search.js` | 每层保留2个剪枝最有效的走法，下一轮优先搜索 |
| **置换表** | `search.js` + `zobrist.js` | 100万条缓存，避免重复搜索相同局面 |
| **静止期搜索** | `search.js` | 到达搜索深度后，若还有活三/冲四威胁，继续延伸搜索（含禁手过滤） |
| **迭代加深** | `search.js` | 从2层逐步加深，时间不够时至少有个次优解 |
| **动态候选限制** | `search.js` | 深度越深候选越少（25→20→15→12→10→**8**），聚焦最强走法 |
| **时间控制** | `search.js` | **18秒**超时保护（地狱难度），防止卡死 |
| **全路径禁手检查** | `search.js` | 开局库/必胜必防/杀棋模块/静止期搜索均覆盖禁手过滤，AI先手不会自投罗网 |
| **多威胁检测** | `evaluate.js` | 自动识别 >=2 个独立冲四威胁，避免只堵一个漏掉另一个 |
| **走法随机化** | `search.js` | 近优走法池随机选取（非必胜/必败局面），每局有自然差异 |

### 杀棋模块架构

```
+-- _detectKill() ----------------------------------------+
|  扫描全盘找到所有活三以上的进攻走法                        |
|  对Top-5最强的走法逐一验证                                |
|  [禁手检查] AI执黑时自动跳过禁手走法(双三/双四/长连)       |
+----------------------+-----------------------------------+
                       |
                       v
+-- _forceKillSearch(attacker, defender, depth) ---------+
|  进攻方视角:                                             |
|  +-- 有冲四 --> 走冲四,切换防守方                         |
|  +-- 有活三 --> 走活三,切换防守方                         |
|  +-- 直接连五 --> 杀棋成立!                               |
|  [禁手检查] 黑棋进攻时跳过禁手走法(双三/双四/长连)          |
|                                                         |
|  防守方视角:                                             |
|  +-- 冲四/活四 --> 唯一应手,必须堵                        |
|  +-- 活三 --> 找所有挡点,尝试是否能全部挡住                |
|  +-- 双冲四 --> 防不住!杀棋成立                           |
|  +-- 所有威胁挡住 --> 杀棋不成立                          |
|                                                         |
|  递归直到深度上限或杀棋确认                               |
+---------------------------------------------------------+
```

## 禁手规则

仅对**黑棋**有效（限制先手优势），可通过复选框开关。

### 禁手类型

| 禁手类型 | 规则 | 示例 |
|---------|------|------|
| 🚫 **双活三禁手** | 一步棋同时形成**两个或以上**的活三 | `_ ● ● ● _`（一个活三，不算禁手）|
| 🚫 **双四禁手** | 一步棋同时形成**两个或以上**的冲四/活四 | `_ ● ● ● ● _`（一个活四，不算禁手）|
| 🚫 **长连禁手** | 一步棋形成**6颗或以上**连续黑子 | `● ● ● ● ● ●` |

### 活三的正确定义

活三特指：**连续3颗同色子，且两端都为空位**，下一步可以形成活四。

```
OK  _ O O O _    --> 活三(两端皆空,下一步成活四)
NG  X O O O _    --> 眠三(一端被堵,下一步只能成冲四)
NG  O O _ O      --> 跳三(中间有空位,下一步只能成冲四)
```

眠三、跳三**不计入**双活三禁手判定。

### 交互体验

判负后会弹出禁手提示窗口，包含两个按钮：
- **查看棋盘** → 关闭弹窗，保留棋局供复盘分析
- **再来一局** → 重新开局

## 开局定式库

收录了五子棋最经典的 **26 种开局 + 9 条深度必胜定式**，覆盖白棋各种应法：

| 开局 | 类型 | 优先级 | 说明 |
|------|------|--------|------|
| **花月**（VCF连杀主线×2） | 黑必胜 | 1（必胜） | 直指斜活三→连续冲四成活四，含白棋常见应法 |
| **浦月**（双杀主线×2） | 黑必胜 | 1（必胜） | 横活三+纵眠三→双三/四杀，多分支覆盖 |
| **雨月**（强化版） | 黑优势 | 1（必胜） | 双活二→活三→VCF连杀 |
| **寒星**（强化版） | 黑优势 | 1（必胜） | 远挡应对→冲四成五 |
| **花月/浦月**（AI执黑专用变例×2） | 黑必胜 | 1（必胜） | 覆盖白2=(8,7)/(7,8)常见防守 |
| 恒星、云月、松月、瑞星等 | 黑优势 | 2（优势） | 标准开局变例 |
| 长星等 | 黑稍优 | 3（可选） | 特殊开局 |

> 地狱难度下优先匹配必胜定式（priority=1），确保AI开局即占据优势。

### AI执白专用策略库

当AI执白后手时，不再依赖现成的开局谱（后者主要面向黑棋），而是使用一套**进攻型策略池**，覆盖 **20个关键局面**：

| 阶段 | 覆盖签名数 | 每局面候选数 | 战术类型 |
|------|-----------|-------------|---------|
| **白2**（应对天元） | 1 | 7种 | standard / aggressive / counterAttack / unorthodox / surprise |
| **白4**（应对黑3） | 8 | 5-6种 | 同上 + trap（陷阱） |
| **白6/白8**（关键分支） | 11 | 4-5种 | 堵中带攻 / 主动反击 / 非常规牵制 |

**随机化机制：**

每个局面下的候选走法都有**不同权重**，采用**加权轮盘赌算法**选取：

```
总权重 = Σweight(i)
rand = random() × totalWeight
遍历候选: rand -= weight → rand ≤ 0 时选中
```

以白2为例（应对黑1天元(7,7)）：

```
(8,8) 经典均衡   ████████████████████████    30%
(8,7) 积极干扰   ██████████████              22%
(7,8) 积极干扰   ██████████████              22%
(6,6) 反向牵制   █████                       10%
(9,9) 非常规     ████                         8%
(6,8) 斜向跳位   ██                           5%
(8,6) 另一侧跳位 ▍                            3%
```

**触发条件：**

- AI执白（后手）
- 当前手数在 **1~12手** 之间（早期阶段）
- 走法位置必须为空
- 优先级仅次于杀棋检测，高于Alpha-Beta搜索

任一触发条件不满足，则回落至搜索引擎。
