# ⚫ 五指棋

一个基于 **Alpha-Beta 剪枝 + 杀棋模块** 的五子棋人机对战游戏，自带 **禁手裁判系统** 和 **棋手评级系统**，所有计算在浏览器本地完成，无需联网。

## 快速开始

1. 克隆或下载本仓库
2. 在浏览器中打开 `index.html`
3. 选择难度和先后手，开始游戏！

## AI 引擎详解

### 决策流程

```
┌────────────────────────────────────────────────────┐
│                   AI 决策流程                        │
├────────────────────────────────────────────────────┤
│                                                     │
│  第1步: 查开局定式库                                 │
│  ├─ 当前局面匹配到棋谱 → 按定式下                    │
│  └─ 没有匹配 → 继续                                 │
│                                                     │
│  第2步: 检查必胜/必防                                │
│  ├─ 有连五/活四 → 立即走棋                          │
│  └─ 对手有活四 → 必须防守                            │
│                                                     │
│  第3步: 杀棋模块检测                                 │
│  ├─ 有活三以上的进攻机会 → 强制搜索16层确认杀棋       │
│  │  ├─ 杀棋成立 → 返回必胜走法                      │
│  │  └─ 杀棋不成立 → 继续                            │
│  └─ 无显著威胁 → 继续                               │
│                                                     │
│  第4步: 迭代加深 Alpha-Beta 搜索                     │
│  ├─ 2层 → 4层 → 6层 → 8层 → 10层（地狱模式）        │
│  ├─ 每层: 走法排序 → Alpha-Beta剪枝 → 置换表缓存     │
│  └─ 达到时间限制 → 返回最佳走法                      │
│                                                     │
└────────────────────────────────────────────────────┘
```

### 搜索优化技术

| 技术 | 文件 | 效果 |
|------|------|------|
| **Alpha-Beta 剪枝** | `search.js` | 跳过无用分支，搜索效率提升数倍 |
| **走法排序** | `search.js` + `evaluate.js` | 进攻分×1.0 + 防守分×1.1 + 位置分，好的排序让剪枝更高效 |
| **杀手走法** | `search.js` | 每层保留2个剪枝最有效的走法，下一轮优先搜索 |
| **置换表** | `search.js` + `zobrist.js` | 100万条缓存，避免重复搜索相同局面 |
| **静止期搜索** | `search.js` | 到达搜索深度后，若还有冲四/活三威胁，继续延伸搜索 |
| **迭代加深** | `search.js` | 从2层逐步加深，时间不够时至少有个次优解 |
| **动态候选限制** | `search.js` | 深度越深候选越少（25→20→15→12→10），聚焦最强走法 |
| **时间控制** | `search.js` | 12秒超时保护（地狱难度），防止卡死 |

### 杀棋模块架构

```
┌─ _detectKill() ────────────────────────────────────┐
│  扫描全盘找到所有活三以上的进攻走法                    │
│  对Top-5最强的走法逐一验证                           │
└──────────┬──────────────────────────────────────────┘
           │ 调用
           ▼
┌─ _forceKillSearch(attacker, defender, depth) ─────┐
│  进攻方视角：                                       │
│  ├─ 有冲四 → 走冲四，切换防守方                      │
│  ├─ 有活三 → 走活三，切换防守方                      │
│  └─ 直接连五 → 杀棋成立！                           │
│                                                     │
│  防守方视角：                                       │
│  ├─ 冲四/活四 → 唯一应手，必须堵                     │
│  ├─ 活三 → 找所有挡点，尝试是否能全部挡住              │
│  ├─ 双冲四 → 防不住！杀棋成立                       │
│  └─ 所有威胁挡住 → 杀棋不成立                        │
│                                                     │
│  递归直到深度上限或杀棋确认                           │
└────────────────────────────────────────────────────┘
```

## 评级系统

对局结束后自动评分，评分因素：

| 因素 | 影响 |
|------|------|
| 胜负 | 胜+500 败不加 |
| 步数效率 | 20步内赢额外+200 |
| 难度系数 | 地狱+500 困难+300 中等+150 |
| 悔棋次数 | 每次-50 |
| 对局复杂度 | 40步以上+100 |

### 段位表

| 段位 | 评分 | 称号 |
|------|------|------|
| 🏆 **S** | ≥ 1700 | 五子棋王者 |
| 💎 **A** | ≥ 1400 | 钻石棋手 |
| 🥇 **B** | ≥ 1150 | 黄金棋手 |
| 🥈 **C** | ≥ 900 | 白银棋手 |
| 🥉 **D** | < 900 | 青铜棋手 |
