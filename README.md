# ⚫ 五指棋

一个基于 **增强型 Alpha-Beta 搜索 + 多维度评估 + 杀棋模块** 的五子棋人机对战游戏，自带 **禁手裁判系统** 和 **棋手评级系统**，所有计算在浏览器本地完成，无需联网。

> **引擎增强**：策略-值双头评估、多精度评估、Pattern增量缓存、空间感知值头、精细化着法分类、VCF/VCT分层搜索

## AI 引擎详解

### 决策流程

```
+----------------------------------------------------------+
|                    AI 决策流程                            |
+----------------------------------------------------------+
|                                                          |
|  Step 1: 查开局定式库(含禁手过滤)                          |
|  +-- 当前局面精确匹配棋谱 --> 秒出应手                     |
|  +-- 不匹配 --> 继续                                      |
|                                                          |
|  Step 1B: AI执白专用策略池(加权随机)                       |
|  +-- 若AI=WHITE 且 手数<=12                               |
|  |   +-- 命中策略池 --> 加权随机选走法,每局不同             |
|  |   +-- 未命中 --> 继续                                   |
|  +-- AI=BLACK或手数>12 --> 跳过                            |
|                                                          |
|  Step 2: 检查必胜/必防(含禁手过滤+多威胁检测)               |
|  +-- 有连五/活四 --> 立即走棋                              |
|  +-- 对手有活四/冲四 --> 必须防守                          |
|  +-- 对手有多重冲四(>=2独立威胁) --> 标记多威胁            |
|  +-- 对手有活三且自己没活四 --> 建议防守                   |
|                                                         |
|  Step 3: 杀棋模块检测(含禁手过滤)                         |
|  +-- 有活三以上的进攻机会 --> 强制搜索16层确认杀棋         |
|  |   +-- 杀棋成立 --> 返回必胜走法                        |
|  |   +-- 杀棋不成立 --> 继续搜索                          |
|  +-- 无显著威胁 --> 继续                                  |
|                                                          |
|  Step 4: 迭代加深 Alpha-Beta 搜索(含禁手过滤+随机化)       |
|  +-- 2层 -> 4层 -> 6层 -> 8层 -> 10层 -> 12层(地狱模式)   |
|  +-- 每层: 走法排序 -> Alpha-Beta剪枝 -> 置换表缓存        |
|  +-- 近优走法随机选取(非必胜/必败局面)                     |
|  +-- 达到时间限制 --> 返回最佳走法                         |
|                                                          |
+----------------------------------------------------------+
```

### 搜索优化技术

| 技术 | 文件 | 效果 |
|------|------|------|
| **Alpha-Beta 剪枝** | `search.js` | 跳过无用分支，搜索效率提升数倍 |
| **走法排序** | `search.js` + `evaluate.js` | 进攻分×1.0 + 防守分×1.1 + 位置分 + importance权重，好的排序让剪枝更高效 |
| **杀手走法** | `search.js` | 每层保留2个剪枝最有效的走法，下一轮优先搜索 |
| **置换表** | `search.js` + `zobrist.js` | 100万条缓存，避免重复搜索相同局面 |
| **静止期搜索** | `search.js` | 到达搜索深度后，若还有活三/冲四威胁，继续延伸搜索（含禁手过滤） |
| **迭代加深** | `search.js` | 从2层逐步加深，时间不够时至少有个次优解 |
| **动态候选限制** | `search.js` | 深度越深候选越少（25→20→15→12→10→8），聚焦最强走法 |

| **VCF/VCT分层搜索** | `search.js` | 叶节点先VCF(冲四链,40层)后VCT(冲四+活三,22层)再静止期，VCF优先利用纯冲四确定性 |
| **极限止损** | `search.js` | 必败检测：对手活四/双冲四/四三杀且己方无反击 → 立即截断，不浪费搜索 |
| **双头评估排序** | `evaluate.js` + `search.js` | 候选走法预计算attackLevel/defendLevel/importance，排序更精准 |

| **Pattern增量缓存** | `evaluate.js` | 搜索过程维持全盘缓存，落子后增量更新半径5格，减少60-70%评估重复调用 |
| **空间感知评分** | `evaluate.js` | 棋盘三维分区CENTER/EDGE/CORNER差异化权重，角落棋型评分打折 |
| **双威胁制造** | `evaluate.js` | 双冲四/四三杀/双活三等必胜棋型大幅加成，AI主动寻找制造多威胁的走法 |
| **盲目冲四惩罚** | `evaluate.js` | 冲四后开放端再远一格也被堵时评分打3折，避免走死棋路 |
| **时间控制** | `search.js` | 18秒超时保护（地狱难度），防止卡死 |
| **全路径禁手检查** | `search.js` | 开局库/必胜必防/杀棋模块/VCF/VCT/静止期搜索均覆盖禁手过滤，AI先手不会自投罗网 |
| **多威胁检测** | `evaluate.js` | 自动识别 >=2 个独立冲四威胁，避免只堵一个漏掉另一个 |
| **间隔棋型识别** | `evaluate.js` | 识别跳冲四(XXX_X)、跳活三(_XX_X_)、跳活二等间隔棋型，评估更精准 |
| **间隔挡点检测** | `search.js` | 防守跳三时自动识别间隔空位及远端挡点，不再遗漏中间挡点 |
| **快速威胁检测** | `search.js` | 叶子节点仅扫描2+连续子附近空位，替代全盘扫描，减少60-70%评估调用 |

## 禁手规则

仅对**黑棋**有效（限制先手优势），可通过复选框开关。

### 禁手类型

| 禁手类型 | 规则 | 示例 |
|---------|------|------|
| 🚫 **双活三禁手** | 一步棋同时形成**两个或以上**的活三 | `_ ● ● ● _`（一个活三，不算禁手）|
| 🚫 **双四禁手** | 一步棋同时形成**两个或以上**的冲四/活四 | `_ ● ● ● ● _`（一个活四，不算禁手）|
| 🚫 **长连禁手** | 一步棋形成**6颗或以上**连续黑子 | `● ● ● ● ● ●` |

### 活三的正确定义

活三特指：**连续3颗同色子，且两端都为空位**，下一步可以形成活四。

```
OK  _ O O O _    --> 活三(两端皆空,下一步成活四)
NG  X O O O _    --> 眠三(一端被堵,下一步只能成冲四)
NG  O O _ O      --> 跳三(中间有空位,下一步只能成冲四)
```

眠三、跳三**不计入**双活三禁手判定。

### 交互体验

判负后会弹出禁手提示窗口，包含两个按钮：
- **查看棋盘** → 关闭弹窗，保留棋局供复盘分析
- **再来一局** → 重新开局

## AI教学模式

勾选控制栏的"教学模式"复选框即可启用，棋盘右侧展开实时讲解面板。

### 落子评级

人类每步落子后，AI 从以下维度评估并给出5级评价：

| 评级 | 图标 | 判定条件 |
|------|------|---------|
| 💡 **妙手** | 金色 | 与AI首选一致 + 形成活四/连五等必胜棋型；或命中AI首选且分数极高 |
| ✅ **好棋** | 绿色 | 与AI首选位置相同，或得分差距 < 5% |
| ⚪ **本手** | 灰色 | 在AI前三候选内，或得分差距 < 20% |
| ⚠️ **缓手** | 橙色 | 得分与AI最佳差距 20%~50%，有更好选择 |
| ❌ **恶手** | 红色 | 得分远低于AI最佳（差距 > 50%），或落子几乎无威胁且有明显更好选点 |

### 面板内容

每步落子后面板实时更新，包含：

| 区域 | 内容 |
|------|------|
| **落子评级** | 评级图标 + 等级文字（妙手/好棋/本手/缓手/恶手） |
| **坐标** | 落子位置，如"黑 H8" |
| **棋型标签** | 识别该步形成的棋型（连五/活四/冲四/活三/眠三/活二/组合棋型） |
| **胜率估算** | 基于 tanh 映射的可视化胜率进度条，1%~99% |
| **局势分析** | 当前全局局势8档评估（压倒性优势→危险） |
| **AI推荐** | 非首选时显示AI推荐的坐标和理由（进攻价值/防守紧迫性） |
| **AI落子解读** | AI落子后显示意图标签（布局/做活三/冲四/必胜等） |

### 胜率估算算法

```
winRate = 0.5 + 0.5 * tanh(advantage / (RUSH_FOUR × 1/phaseFactor))

phaseFactor = 0.6 + 0.4 * min(棋子数 / 30, 1)
```

- 优势为0 → 胜率50%（均势）
- 优势=冲四级别 → 胜率~76%（明显优势）
- 优势=双冲四级别 → 胜率~88%（压倒性优势）
- 棋局阶段缩放：开局评估保守（更靠近50%），残局评估更敏感
- 胜率以彩色进度条+百分比数字展示（绿>黄>红）

### 交互设计

- **默认折叠**：面板初始折叠为竖排标签"📖 AI 讲解"，不遮挡棋盘
- **勾选展开**：勾选"教学模式"后面板自动展开
- **手动折叠**：点击面板标题栏可随时折叠/展开，兼顾沉浸对弈
- **响应式布局**：窄屏（≤900px）面板移至棋盘下方

## 开局定式库

收录了五子棋最经典的 **26 种开局 + 9 条深度必胜定式**，覆盖白棋各种应法：

| 开局 | 类型 | 优先级 | 说明 |
|------|------|--------|------|
| **花月**（VCF连杀主线×2） | 黑必胜 | 1（必胜） | 直指斜活三→连续冲四成活四，含白棋常见应法 |
| **浦月**（双杀主线×2） | 黑必胜 | 1（必胜） | 横活三+纵眠三→双三/四杀，多分支覆盖 |
| **雨月**（强化版） | 黑优势 | 1（必胜） | 双活二→活三→VCF连杀 |
| **寒星**（强化版） | 黑优势 | 1（必胜） | 远挡应对→冲四成五 |
| **花月/浦月**（AI执黑专用变例×2） | 黑必胜 | 1（必胜） | 覆盖白2=(8,7)/(7,8)常见防守 |
| 恒星、云月、松月、瑞星等 | 黑优势 | 2（优势） | 标准开局变例 |
| 长星等 | 黑稍优 | 3（可选） | 特殊开局 |

> 地狱难度下优先匹配必胜定式（priority=1），确保AI开局即占据优势。

### AI执白专用策略库

当AI执白后手时，不再依赖现成的开局谱（后者主要面向黑棋），而是使用一套**进攻型策略池**，覆盖 **20个关键局面**：

| 阶段 | 覆盖签名数 | 每局面候选数 | 战术类型 |
|------|-----------|-------------|---------|
| **白2**（应对天元） | 1 | 7种 | standard / aggressive / counterAttack / unorthodox / surprise |
| **白4**（应对黑3） | 8 | 5-6种 | 同上 + trap（陷阱） |
| **白6/白8**（关键分支） | 11 | 4-5种 | 堵中带攻 / 主动反击 / 非常规牵制 |

**随机化机制：**

每个局面下的候选走法都有**不同权重**，采用**加权轮盘赌算法**选取：

```
总权重 = Σweight(i)
rand = random() × totalWeight
遍历候选: rand -= weight → rand ≤ 0 时选中
```

以白2为例（应对黑1天元(7,7)）：

```
(8,8) 经典均衡   ████████████████████████    30%
(8,7) 积极干扰   ██████████████              22%
(7,8) 积极干扰   ██████████████              22%
(6,6) 反向牵制   █████                       10%
(9,9) 非常规     ████                         8%
(6,8) 斜向跳位   ██                           5%
(8,6) 另一侧跳位 ▍                           3%
```

**触发条件：**
- AI执白（后手）
- 当前手数在1~12手之间（早期阶段）
- 走法位置必须为空
- 优先级仅次于杀棋检测，高于Alpha-Beta搜索

任一触发条件不满足，则回落至搜索引擎。
